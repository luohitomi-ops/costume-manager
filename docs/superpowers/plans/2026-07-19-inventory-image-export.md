# Inventory Image Export Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** a "下載所有服裝道具圖片" button on `index.html` that generates a single nicely formatted PNG of the user's entire inventory (grouped by character), downloaded straight to the browser — so it can be sent to a friend over LINE/Messenger without a live link, and works identically whether the user is on the local dev server or her deployed Vercel cloud instance.

**Architecture:** This deviates from the original design doc's server-side-Playwright approach (`docs/superpowers/specs/2026-07-18-distribution-and-access-design.md`, Project 2 section). That approach assumed a Playwright-driven headless Chromium render would work identically in both environments — it does not: standard Playwright ships a full Chromium binary that does not run in Vercel's serverless function environment without extra packages (`@sparticuz/chromium` or similar) and careful size-budget management, which the design doc never accounted for (Project 3's cloud deployment work happened after this design doc was approved). Since the user just found this feature missing specifically on her live cloud instance, a server-rendered approach re-opens exactly that risk.

Instead: **capture happens entirely client-side**, in the user's own browser, using the [html2canvas](https://html2canvas.hertzen.com/) library (vendored locally, no CDN dependency — this project runs fully offline when self-hosted). The existing `GET /api/export` endpoint already returns `{ characters, items }` (items already joined with `character_name`, ordered by character then item name — see `src/routes/export.js` and `src/models/item.js`'s `allItemsWithCharacters()`). Clicking the button fetches that JSON, renders it into an off-screen styled container already present in the page, and `html2canvas` snapshots that container to a `<canvas>`, which is then downloaded as a PNG via a synthetic `<a download>` click. No new backend route, no server-side rendering, no dependency on `DB_DRIVER` — identical code path locally and on Vercel.

**Tech Stack:** vanilla browser JS (matches every other page in `public/`, no bundler), `html2canvas` 1.4.1 (new dependency, vendored — not loaded from a CDN), Playwright (already a devDependency) for this plan's own verification only.

## Global Constraints

- No new backend routes or model changes — `GET /api/export` already returns everything this feature needs. If you find yourself adding a route, stop and re-read this plan; that means something above was misread.
- No CDN script tags. `html2canvas` must be vendored into `public/vendor/` and regenerated automatically on `npm install` (this project runs fully offline for local self-hosters — see README's "Option A" — a CDN dependency would silently break that).
- Must work with no code branching on `DB_DRIVER` or `ACCESS_PASSWORD` — the whole point of the client-side approach is that it doesn't care which mode the server is in.
- Follow this codebase's existing convention of not HTML-escaping user-entered strings before inserting via `innerHTML` (see `public/index.html`'s existing `refreshCharacterDropdowns` and `shared.js`'s `itemToLine`) — this is a single-password, non-multi-tenant local tool, and introducing escaping in only one new code path while the rest of the app has none would be inconsistent, not safer.
- No test framework exists in this project — verification is a manual Playwright script, written and deleted per-task, matching `specs/003-category-management/plan.md`'s convention.

- [ ] **Step 2: Write the copy script**

```js
import fs from 'node:fs';
import path from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = path.dirname(fileURLToPath(import.meta.url));
const root = path.join(__dirname, '..');

const vendorDir = path.join(root, 'public', 'vendor');
fs.mkdirSync(vendorDir, { recursive: true });

fs.copyFileSync(
  path.join(root, 'node_modules', 'html2canvas', 'dist', 'html2canvas.min.js'),
  path.join(vendorDir, 'html2canvas.min.js')
);

console.log('Copied html2canvas.min.js to public/vendor/');
```

- [ ] **Step 3: Wire it into `postinstall` so it's regenerated automatically**


This means: every `npm install` (including the one Vercel runs at deploy time, and the one a friend running the offline `.exe` build machine would have already run) regenerates `public/vendor/html2canvas.min.js` — it never needs to be committed, same reasoning as `node_modules/` itself.

- [ ] **Step 4: Add the render + capture + download script**

Find the end of the existing `<script>` block:
```js
loadCategoryOptions();
refreshCharacterDropdowns();
```

Add immediately after those two lines:
```js
function renderExportCapture(characters, items) {
  document.getElementById('export-date').textContent = `匯出時間：${new Date().toLocaleString('zh-TW')}`;

  const itemsByCharacter = new Map();
  for (const item of items) {
    if (!itemsByCharacter.has(item.character_id)) itemsByCharacter.set(item.character_id, []);
    itemsByCharacter.get(item.character_id).push(item);
  }

  const groupsHtml = characters
    .map((character) => {
      const characterItems = itemsByCharacter.get(character.id) || [];
      if (characterItems.length === 0) return '';
      return `
        <div style="margin-bottom:20px;">
          <h2 style="font-size:1.1rem;font-weight:700;color:#3F5C46;margin:0 0 8px;padding-bottom:4px;border-bottom:2px solid #E4DBC5;">${character.name}</h2>
          <ul style="list-style:none;margin:0;padding:0;display:flex;flex-direction:column;gap:6px;">
            ${characterItems.map((item) => itemToLine(item, categoryLabels)).join('')}
          </ul>
        </div>
      `;
    })
    .join('');

  document.getElementById('export-groups').innerHTML =
    groupsHtml || '<p style="text-align:center;color:#8B8374;">尚未登記任何服裝道具</p>';
}

document.getElementById('export-image-button').addEventListener('click', async () => {
  const btn = document.getElementById('export-image-button');
  const originalLabel = btn.textContent;
  btn.disabled = true;
  btn.textContent = '產生圖片中…';
  try {
    const { characters, items } = await fetch('/api/export').then((r) => r.json());
    renderExportCapture(characters, items);
    const canvas = await html2canvas(document.getElementById('export-capture'), {
      backgroundColor: '#FBF8F1',
      scale: 2,
    });
    const link = document.createElement('a');
    link.download = `服裝道具清單-${new Date().toISOString().slice(0, 10)}.png`;
    link.href = canvas.toDataURL('image/png');
    link.click();
  } catch (err) {
    alert('產生圖片失敗，請稍後再試。');
  } finally {
    btn.disabled = false;
    btn.textContent = originalLabel;
  }
});
```

```js
import { chromium } from 'playwright';
import fs from 'node:fs';

const browser = await chromium.launch();
const page = await browser.newPage({ viewport: { width: 1280, height: 900 } });

const consoleErrors = [];
page.on('console', (msg) => { if (msg.type() === 'error') consoleErrors.push(msg.text()); });
page.on('pageerror', (err) => consoleErrors.push(String(err)));

await page.goto('http://localhost:3000', { waitUntil: 'networkidle' });

const [download] = await Promise.all([
  page.waitForEvent('download'),
  page.click('#export-image-button'),
]);

const savePath = '/tmp/verify-image-export.png';
await download.saveAs(savePath);

const stats = fs.statSync(savePath);
const buf = fs.readFileSync(savePath);
const isPng = buf[0] === 0x89 && buf[1] === 0x50 && buf[2] === 0x4e && buf[3] === 0x47;

console.log('downloaded file size:', stats.size, 'bytes');
console.log('valid PNG signature:', isPng);
console.log('console errors:', consoleErrors.length);
if (consoleErrors.length) console.log(consoleErrors.join('\n'));

await browser.close();
```

## Deploying this to the live cloud instance

This plan only changes `public/index.html` (pure static asset, served as-is) plus the `postinstall`-generated `public/vendor/html2canvas.min.js` — no server code, no schema, no env vars. Once both tasks are committed, redeploy the existing Vercel project the same way it was originally deployed (`npx vercel --prod` from the repo root, or push if Vercel's GitHub integration is connected) to make the button appear on the live cloud instance. No migration script, no new environment variables, no schema change needed.
