# 服裝道具管家 — 安裝教學

[README](README.zh-TW.md) 是給比較熟電腦的人看的精簡版，這篇是完整、每一步都寫清楚的版本，
假設你旁邊沒有 AI 可以問，照著做應該就能自己裝起來。

有兩種玩法：

- **方案 A：本地自架** — 只要會用「複製貼上」跟打幾行指令，難度低，資料存在你自己電腦裡。
- **方案 B：部署你自己的雲端版** — 手機隨時能查，但需要申請兩個帳號、多跑幾個步驟，難度中等。

建議第一次先裝方案 A，確定能跑起來、玩得順手之後，有需要再挑戰方案 B。

---

## 方案 A：本地自架

### 第 1 步：安裝 Node.js

這個工具是用 Node.js 寫的，你的電腦需要先裝好它才能執行。

1. 打開瀏覽器，到 [nodejs.org](https://nodejs.org)
2. 網站會自動偵測你的作業系統，點擊寫著 **LTS**（長期支援版）字樣的那個綠色下載按鈕
   （不要選 Current／最新版，LTS 比較穩定）
3. 下載完成後雙擊安裝檔，全部選項都按「下一步」用預設值就好，一路按到「安裝」「完成」
4. **安裝完成後，如果你有開著的視窗（尤其是命令提示字元/終端機），先全部關掉**——這一步很多人漏掉，
   之後會出現「找不到指令」的錯誤

**怎麼確認裝成功了：**

- **Windows**：按 `Win` 鍵，打「cmd」，按 Enter 打開「命令提示字元」
- **Mac**：按 `Cmd + 空白鍵`，打「terminal」，按 Enter 打開「終端機」

打開後，輸入這行指令再按 Enter：

```
node --version
```

如果看到類似 `v22.13.0` 這樣的版本號，就是裝好了。如果出現「'node' 不是內部或外部命令」
（Windows）或「command not found」（Mac），代表沒裝成功或視窗沒重開，回到步驟 1 重新確認一次。

### 第 2 步：下載程式碼

不需要額外裝任何東西，直接在網頁上下載：

1. 打開 [github.com/luohitomi-ops/costume-manager](https://github.com/luohitomi-ops/costume-manager)
2. 找到綠色的 **Code** 按鈕，點下去
3. 選 **Download ZIP**
4. 下載完成後，找到那個壓縮檔（通常在「下載」資料夾），**解壓縮**它
   （Windows 右鍵點壓縮檔選「全部解壓縮」；Mac 直接雙擊）
5. 解壓縮後會得到一個資料夾，名字類似 `costume-manager-master`，記住它放在哪裡
   （建議直接放在桌面，等一下比較好找）

### 第 3 步：打開終端機，切換到這個資料夾

繼續用第 1 步打開的命令提示字元／終端機視窗（沒關的話可以繼續用；關掉了就照第 1 步的方法重新打開一個）。

**最簡單的做法**：打開資料夾總管（Windows 檔案總管 / Mac Finder），找到剛剛解壓縮出來的
`costume-manager-master` 資料夾：

- **Windows**：打開這個資料夾後，點網址列（顯示路徑的那一條），把裡面的文字全部清掉，
  改打 `cmd` 再按 Enter，會直接跳出一個已經在這個資料夾裡的命令提示字元視窗
- **Mac**：在 Finder 裡選到這個資料夾，直接把資料夾整個拖進 Terminal 視窗，會自動貼上完整路徑，
  再手動打 `cd ` （cd 後面有一個空格）貼在最前面，按 Enter

打完之後，輸入以下指令確認自己真的在對的資料夾裡：

```
dir
```
（Mac 用 `ls`）

如果看到 `package.json`、`public`、`src` 這些檔案/資料夾名稱，就代表對了，可以繼續下一步。

### 第 4 步：安裝相依套件

在同一個視窗打這行，按 Enter：

```
npm install
```

會跑一陣子（可能幾十秒到幾分鐘，看網路速度），跑完會回到可以打字的狀態，中間出現幾行黃色的
`npm warn` 字樣是正常的，不用理它。**如果整個視窗最後印出紅色的 `npm error` 而且停住**，
把那段錯誤訊息記下來（截圖也行），這是唯一比較需要進一步排查的地方。

### 第 5 步：啟動

打這行，按 Enter：

```
npm run dev
```

看到這一行就代表成功了：

```
Costume Manager running at http://localhost:3000
```

**注意：這個視窗不能關**，關掉程式就會停止運作。之後每次要用，都要重複第 3~5 步
（打開終端機 → 切到資料夾 → `npm run dev`），或是乾脆把這個視窗縮小放著。

### 第 6 步：打開來用

打開瀏覽器（Chrome、Edge、Safari 都可以），網址列輸入：

```
localhost:3000
```

按 Enter，應該就會看到「服裝道具管家」的畫面了。資料存在你自己電腦裡的一個檔案
（`data/costume-manager.db`），不會傳到任何地方，不需要密碼，也不需要連網。

## 方案 B：部署你自己的雲端版

這個方案能讓你不開電腦、用手機隨時查你的庫存。步驟比方案 A 多，但都是照著複製貼上跑指令，
不需要真的懂程式。**建議先完成方案 A、確定基本操作沒問題後再來挑戰這個。**

**⚠️ Windows 使用者請先看這段**：這個方案需要用到 Turso 這個服務的指令工具，但它目前**沒有
Windows 版**，只能透過「WSL」（Windows 內建的 Linux 環境）使用。以下步驟已經把 WSL 的安裝方法
寫進去了，跟著做即可，但這會比方案 A 多花一點時間跟一次重開機。如果不想裝 WSL，也可以跳過
第 4 步的指令，改到 [app.turso.tech](https://app.turso.tech) 網頁後台手動建立資料庫、
在資料庫的頁面裡找「連線網址／Connect」跟「建立 Token／Create Token」拿到需要的兩個值——網頁介面
偶爾會改版，如果找不到對應按鈕，可以到 [docs.turso.tech](https://docs.turso.tech) 查最新畫面。
Mac 使用者不受影響，直接往下做就好。

### 需要先申請的東西

1. **Vercel 帳號**（免費）：去 [vercel.com](https://vercel.com)，用 GitHub 或 Email 註冊登入
2. **Turso 帳號**（免費）：去 [turso.tech](https://turso.tech)，一樣註冊登入

### 第 1 步：完成方案 A 的第 1~4 步

雲端版的程式碼跟方案 A 是同一包，所以要先照上面「方案 A」的第 1 步到第 4 步做過一次
（裝 Node.js、下載程式碼、`npm install`），確定能跑 `npm run dev` 打開網頁沒問題，再繼續下面的步驟。

### 第 2 步：安裝兩個額外的指令工具

**Vercel 的部分，Windows / Mac 都一樣**，在終端機（要在 `costume-manager` 這個資料夾裡）打：

```
npm install -g vercel
```

**Turso 的部分，Mac 使用者**直接在同一個終端機打：

```
curl -sSfL https://get.tur.so/install.sh | bash
```

**Turso 的部分，Windows 使用者**因為沒有原生 Windows 版，需要先裝 WSL：

1. 按 `Win` 鍵打「powershell」，**右鍵點「Windows PowerShell」選「以系統管理員身分執行」**
   （一般直接打開的不算，一定要選這個「系統管理員」的選項）
2. 在跳出來的藍底視窗打這行，按 Enter：
   ```
   wsl --install
   ```
3. 跑完後會要求**重新開機**，存檔關掉其他東西後就重開機
4. 重開機後，按 `Win` 鍵打「Ubuntu」打開它（第一次打開會要你設一組使用者名稱跟密碼，
   自己設一組記得住的就好，這組密碼只有你自己這台電腦用得到）
5. 在這個 Ubuntu 視窗（不是原本的命令提示字元）裡，打：
   ```
   curl -sSfL https://get.tur.so/install.sh | bash
   ```

**之後每次要用到 Turso 指令，都要重新打開「Ubuntu」這個視窗來打指令**（不是命令提示字元），
下面幾步凡是跟 `turso` 開頭的指令都在這個 Ubuntu 視窗裡打；跟 `vercel`、`npm`、`node` 開頭的指令，
繼續在原本的命令提示字元／終端機視窗打。

（如果不想裝 WSL，可以改到 [app.turso.tech](https://app.turso.tech) 網頁後台手動操作，
跳過第 2 步跟第 4 步的 turso 指令，直接看上面「⚠️ Windows 使用者請先看這段」的說明）

### 第 3 步：登入這兩個工具

`vercel login`（Windows 使用者在原本的命令提示字元打；Mac 使用者在終端機打）：

```
vercel login
```
會跳出提示，選 email 登入或直接開瀏覽器授權，跟著畫面指示做完。

`turso auth login`（**Windows 使用者這行要在「Ubuntu」視窗裡打，不是命令提示字元**；
Mac 使用者一樣在終端機打）：

```
turso auth login
```
一樣會開瀏覽器要你授權，授權完回到視窗會顯示登入成功。

### 第 4 步：建立你自己的資料庫

**Windows 使用者：接下來這整個第 4 步都在「Ubuntu」視窗裡打**（Mac 使用者在終端機打）：

```
turso db create costume-manager
```

跑完後，打這兩行分別拿到兩個等一下要用的值：

```
turso db show costume-manager --url
```
這行印出來的網址，等一下要用。

```
turso db tokens create costume-manager
```
這行印出來的一長串文字（token），等一下也要用——**這串很長很像亂碼是正常的，直接整段複製**。

### 第 5 步：設定環境變數檔案

在 `costume-manager` 資料夾裡，找到 `.env.example` 這個檔案，**複製一份**、把複製出來的檔名改成
`.env.local`（Windows 右鍵「複製」再「貼上」，然後幫貼上的那份改名）。

用記事本（或任何文字編輯器）打開 `.env.local`，會看到：

```
DB_DRIVER=local
TURSO_DATABASE_URL=
TURSO_AUTH_TOKEN=
ACCESS_PASSWORD=
```

改成這樣（把等號後面填上剛剛拿到的值）：

```
DB_DRIVER=turso
TURSO_DATABASE_URL=（貼上第4步 turso db show 印出來的網址）
TURSO_AUTH_TOKEN=（貼上第4步 turso db tokens create 印出來的一長串）
ACCESS_PASSWORD=（自己想一組密碼，這是你以後打開雲端版要輸入的密碼，只有你自己知道）
```

存檔。

### 第 6 步：建立資料表

**回到原本的命令提示字元／終端機視窗**（不是 Ubuntu，這一步是跑專案裡的 Node.js 程式，
要在 `costume-manager` 資料夾那個視窗打）：

```
node scripts/setup-turso-schema.mjs
```

看到最後印出 `Done. Turso schema ready.` 就是成功了。

### 第 7 步（選用）：把本地資料搬過去

如果你已經用方案 A 登記了一些角色跟道具，想把這些資料一起帶到雲端版，打：

```
node scripts/migrate-to-turso.mjs
```

如果不想搬、想從空的開始，這步可以跳過。

### 第 8 步：部署到 Vercel

**這一步開始都回到原本的命令提示字元／終端機視窗**（不是 Ubuntu）：

```
vercel link
```
會問幾個問題，一路按 Enter 用預設值就好（如果問要不要連結現有專案，選「否」建立新的）。

接下來這四行，**每一行按 Enter 後都會再問你要貼哪個值**，把 `.env.local` 裡對應的那一行貼上去：

```
vercel env add TURSO_DATABASE_URL production
```
```
vercel env add TURSO_AUTH_TOKEN production
```
```
vercel env add ACCESS_PASSWORD production
```
```
vercel env add DB_DRIVER production
```
（最後這個，被問到要貼什麼值時，直接打 `turso`）

全部設定完後，打：

```
vercel --prod
```

跑完會印出一個網址，長得像 `https://costume-manager-xxxx.vercel.app`，這就是你的雲端版網址了。

### 第 9 步：打開來用

用手機或電腦瀏覽器打開上一步印出來的網址，會先看到一個要求輸入密碼的畫面，輸入你在第 5 步
自己設定的 `ACCESS_PASSWORD`，就能進去了。之後可以把這個網址存到手機主畫面，隨時打開用。

**這組密碼只有你自己知道，不要分享給別人**——分享出去等於讓對方能看到、修改、刪除你的資料。

## 常見問題

**Q：`npm` 或 `node` 指令說「找不到」，明明我裝過 Node.js 了？**
把所有終端機視窗都關掉，甚至整台電腦重開機，再重新打開一個新的命令提示字元試一次。
Windows 常常要重開機或至少關掉舊視窗，新裝的指令才會生效。

**Q：`npm install` 跑到一半卡住或報錯？**
確認網路連線正常，重新打一次 `npm install` 試試看。如果同樣的錯誤訊息一直出現，
把錯誤訊息複製起來，貼給任何一個 AI（ChatGPT、Claude 都可以）問它，通常就能問出原因。

**Q：`localhost:3000` 打不開／顯示「無法連上這個網站」？**
確認 `npm run dev` 那個視窗還開著、還顯示 `Costume Manager running at...` 沒有跳出錯誤，
如果那個視窗被關掉了，回到程式資料夾重新打一次 `npm run dev`。

**Q：方案 B 忘記密碼怎麼辦？**
打開 `.env.local` 看 `ACCESS_PASSWORD` 那一行就是你設的密碼；如果連本地端的檔案都不見了，
用 `vercel env ls` 可以查目前雲端上設定了哪些變數名稱（但看不到實際的值），
這時候最簡單的方式是重新跑一次「第 5 步」設一組新密碼，再用
`vercel env rm ACCESS_PASSWORD production` 刪掉舊的、`vercel env add ACCESS_PASSWORD production`
加新的，然後 `vercel --prod` 重新部署一次。
